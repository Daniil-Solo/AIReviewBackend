# Критерии оценки кода проекта с использованием API

## Когда применяется

Если проект связан с написанием API

## Критерии

### URI должны обозначать ресурсы, (существительные), а не операции над ними (операции определяются HTTP-методами)

Примеры:
✅ GET /users
❌ GET /getUsers, POST /createUser


### В URI используется множественное число сущности, если можно получить список или конкретный объект

Примеры:
✅ GET /users/123, POST /orders
❌ GET /user/123, POST /order


### Иерархия сущностей должна быть отражена в URI

Примеры:
✅ GET /users/123/orders
❌ GET /orders?userId=123


### В URI не должно быть глаголов действия (есть исключения)

Для указания действия используются HTTP-методы (GET, POST, PUT, PATCH, DELETE).

Примеры:
✅ DELETE /users/123
❌ GET /users/123/delete

Исключения (нестандартные операции):
POST /users/123/activate


### Фильтрация, сортировка и пагинация должна выполняться через Query-параметры

Для настройки коллекций используйте query-параметры (после ?), а не расширение пути.

Примеры:
✅ GET /users?role=admin&page=2&sort=createdAt_desc
❌ GET /users/admin/page/2


### Именование сущностей в URI должно быть единообразным в рамках всего API

Должен быть один стиль написания составных имён (kebab-case, snake_case, camelCase) во всех URI.

✅ /user-profiles, /order-items
❌ /userProfiles, /order_items (два разные стиля написания)


### Правильно используются стандартные HTTP-коды ответов

200 OK — успешный GET, PUT, PATCH, DELETE.
201 Created — успешный POST (создание).
204 No Content — успешный DELETE или PUT без тела ответа.
400 Bad Request — проблема с запросом.
401 Unauthorized — пользователь не аавторизован
403 Forbidden — недостаточно прав (действие запрещено)
404 Not Found — ресурс не найден.
409 Conflict — конфликт (например, дубликат).
422 Unprocessable Entity — ошибка валидации.
