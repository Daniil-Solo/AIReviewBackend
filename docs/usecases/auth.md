## Регистрация, аутентификация

### Регистрация

Форма регистрации: 
- Полное имя (fullname)
- Электронная почта (email)
- Пароль и повтор пароля (password)

После успешного заполнения:
- форма сменяется на окно ввода кода
- на указанную почту отправляется код

Примечание: код доступен в течение 60 секунд, после чего он станет недействительным

После ввода верного кода:
- регистрация завершается
- пользователь перенаправляется на страницу личного кабинета

Техническая реализация:
- после отправки формы регистрации в базе данных создается запись со флагом is_verified=False
- пароль должен быть захэширован с помощью hashlib.pbkdf2_hmac
- код хранится в Redis с временем жизни 60 секунд
- после подтверджения по коду из письма пользователь в базе становится is_verified=True
- после завершения регистрации пользователь получает сессионный токен доступа с временем жизни 12 часов
- информация о пользователе хранится в Redis: ключ - сессионный токен, значение - информация о пользователй {id, fullname, email}

Риски:
- в базе данных может быть много неподтвержденных записей, которые придется периодически чистить

Эндпоинты:
- `POST /auth/register`: Request: {fullname, email, password}, Response {message}
- `POST /auth/verify`: Request {email, code}, Response {access_token}


### Аутентификация

Форма входа:
- Электронная почта (email)
- Пароль (password)

После ввода верного кода:
- пользователь перенаправляется на страницу личного кабинета

Эндпоинты:
- `POST /auth/login`: Request {email, password}, Response {access_token}


### Логаут

При нажатии на кнопку "Выйти" пользователь выходит из своего аккаунта и оказывается на странице со входом

Техническая реализация:
- удаляется информация о пользователе из Redis

Эндпоинты:
- `POST /auth/logout`: Response {message}
