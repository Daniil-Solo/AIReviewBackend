# Использование систем искусственного интеллекта и личный вклад автора

## Разработка и написание исходного кода системы

В рамках данной работы для ускорения написания исходного кода применялся OpenCode. 
Это открытый инструмент для программирования в агентном режиме с помощью больших языковых моделей.
В качестве моделей использовались облачные модели семейств Qwen, MiniMax, GLM,  которые показывают высокий
уровень решения задач по программированию среди открытых моделей.

Для разработки серверной части перед взаимодействием с агентом были вручную автором написаны артефакты системы:
архитектура сервиса в нотации C4 (уровень контейнеров), модель данных в виде ER (Entity-Relationship) диаграммы,
текстовые описания сценариев использования системы с акцентом на закладываемые технические решения, архитектура
работы пайплайна для оценивания работы в виде диаграммы потока данных. Перечисленные верхнеуровневые артефакты
в дальнейшем помогали агенту для реализации новых возможностей в системе. 

В процессе взаимодействия с агентом итеративно дополнялся файл AGENTS.md, предназначенный для наполнения исходного 
контекста агента при разработки новых возможностей. В этот файл была включена основная информация о проекте, 
ссылки на артефакты и инструкции по реализации типичных фрагментов кода (схемы данных, сервисные функции, эндпоинты, 
инъекция зависимостей, тесты). Основная задача была в том, чтобы ограничить агента и заставить писать код в ожидаемом
для автора формате.

Для релизации клиентской части также был подготовлен файл AGENTS.md, включающий основную информацию о проекте, 
используемые технологии и правила написания кода. Дополнительно был сгенерирована OpenAPI-документация доступных
на сервере эндпоинтов и приведена к более емкому и читаемому формату, чтобы агент мог всегда обратиться к существующим
эндпоинтам системы. Ввиду недостатка опыта в разработке клиентской части приложения
не удалось на старте взаимодействия с агентом задать четкие инструкции по реализации типичных фрагментов кода, из-за
чего агент иногда генерировал для похожих сценариев различный код.

Реализация новой возможности с помощью агента состояла в том, что автор отправлял ему описание, пути до файлов, которые
могут быть полезны и в режиме диалога просил внести агента правки по разработанному коду. В некоторых случаях агент
начинал ходить кругами в процессе устранения бага и не мог справиться с задачей, тогда автор приступал к ручной отладке
и самостоятельному поиску решения проблемы. В основном ошибки были в процессе разработки клиентской части.

Результат работы агента просматривался автором лично и корректировался в зависимости от собственных соображений по разработке. 
Тестированием всех сценариев использования системы автор занимался лично. Выбор провайдеров для доступа к большим языковым моделям, 
проведения платежей и рассылки почтовых уведомлений также принадлежит автору.


## Коррекция стиля текста для ВКР

Для приближения к более официальному стилю использовалась большая языковая модель DeepSeek с системным промптом.
Системный промпт содержал правила обработки теста с точки зрения стиля, структуры, грамматики, логики, академической этики.

Автор гарантирует, что исходные мысли принадлежат ему и не искажены в результате обработки большой языковой моделью.
Каждый перегенерированный фрагмент текста был проверен вручную автором.
