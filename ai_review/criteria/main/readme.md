## Критерии

### Task-Specific

- соответствие поставленному тз


### Архитектура (architecture)

- выбор компонентов (достаточно, не стреляем ли мы по воробьям из пушки)


### Документируемость (documentation)

- комментарии


### Масштабируемость (scalability)


### Поддерживаемость (maintainability)

- расширяемость системы
- есть скрипты для запуска приложения, наполнения данными
- наличие тестов (разнообразных)

### Читаемость (readability)

- структура кода проекта
- нейминг функций, классов

### Безопасность (security)
